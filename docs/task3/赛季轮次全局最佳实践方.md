# 赛季轮次全局最佳实践方案（不向下兼容）

## 目标与约束
- 每个轮次必须有“总时长上限”，不能无限进行
- 三步流程严格顺序：大纲 → 章节 → 评论（+ 收尾检测）
- 只补漏**大纲与章节**，不补评论
- 评论只针对“当轮新章节”
- 不增加独立“追补阶段”，补漏内化在流程里

## 一、轮次时长与阶段规则（硬约束）
1. 每轮配置 `roundDuration`（分钟），轮次开始记录 `roundStartTime`
2. 轮次结束条件二选一：
   - **完成结轮**：四步全部成功完成
   - **到时结轮**：达到 `roundDuration`，即使四步未完成也必须结束本轮
3. HUMAN_READING = `roundDuration - AI 实际耗时`，允许为 0（最少阅读时长可为 0）
4. 到时结轮时，不强制中断正在执行的任务，但需标记“本轮已到时”，任务结束后做结轮收尾
5. 轮次内三步严格串行，本轮未完成不得开始下一轮对应步骤
6. 轮次之间严格串行，上一轮未完成不得启动下一轮任何步骤

---

## 二、三步顺序与职责边界
### 步骤1：大纲生成
- 第1轮：生成整本书大纲（从零开始）
- 第2轮起：读取上一章评论 + 作者性格（adaptability），判断是否修改大纲
- 修改范围仅限“当前轮及之后章节”

### 步骤2：章节生成
- 第N轮只生成第N章内容
- 依赖该章节大纲，如大纲缺失先补大纲（见补漏规则）

### 步骤3：评论生成
- 仅生成**当前轮新章节**的评论
- 不回补历史评论
- 评论缺失不构成“补漏”触发条件

### 收尾检测
- 只检测“章节是否落后”，不检测评论
- 发现落后 -> 进入补漏流程（仅大纲/章节）

---

## 三、补漏规则（仅大纲与章节）
### 触发时机
- 在**新一轮开始前**执行补漏检查
- 或在本轮步骤执行中发现缺口时即时补（幂等）

### 补漏范围
- **补大纲**：缺失该章节大纲则补
- **补章节**：缺失该章节内容则补
- **不补评论**：评论只在章节当轮触发一次

---

## 四、失败处理与重试策略
1. 每步记录状态：未开始 / 进行中 / 成功 / 失败
2. 失败允许重试（有上限）
3. 达到重试上限后：
   - 若未到轮次总时长 → 继续重试直到到时结轮
   - 若已到轮次总时长 → 进入“到时结轮”，并将缺口放到下一轮补
4. 到时后的补跑仍受最大重试次数限制，超过上限直接标记缺口并结束本轮

---

## 五、推进与结算逻辑（统一规则）
- 进入 AI_WORKING：创建本轮任务，记录 aiWorkStartTime
- 任务完成 → 结轮 → 进入 HUMAN_READING（剩余时间）
- 到时未完成 → 仅标记到时，不进入下一轮，继续完成本轮任务
- 到时结轮不强制中断任务，但任务完成后必须触发“统一收尾动作”，补齐缺口统计与结轮标记
- HUMAN_READING 结束 → 进入下一轮 AI_WORKING
### 到时未完成的处理规则
- 轮次到时立即写入 timedOutAt 与轮次状态=TIMED_OUT，并记录“未完成步骤清单”
- 不撤销或强杀正在执行的任务，但到时后不启动任何下一轮步骤或任务
- 到时后的本轮任务必须继续跑完，完成后才允许进入下一轮
- 任务结束时必须触发统一收尾：补齐缺口、标记本轮完成、清理未完成步骤清单

---

## 六、核心一致性原则
- 轮次有硬上限
- 三步严格顺序 + 收尾检测
- 补漏只针对大纲与章节
- 评论只在当轮新章节触发
- 失败不会静默跳过，只能因“到时结轮”推进

## 七、执行顺序建议（落地顺序）
1. 统一“轮次完成/到时完成”模型与状态
2. 调整四步执行，嵌入幂等补漏（仅大纲/章节）
3. 评论逻辑固定为“当轮新章节一次性触发”
4. 轮次到时结轮机制与收尾逻辑
5. 失败重试与缺口标记规范化

## 修改点总览（对应问题清单）
1. 建立“完成结轮/到时结轮”双路径，禁止时间推进绕过未完成的三步与收尾检测
2. 统一到时结轮的收尾规则：标记缺口、进入阅读期、下一轮补漏
3. 明确评论不补漏，只在当轮新章节触发一次
4. 补漏只针对大纲与章节，并固定触发时机（新一轮开始前 + 发现缺口即时补）
5. 失败与重试的统一策略：到时结轮后把缺口延续到下一轮补
6. 到时结轮后的任务完成必须触发统一收尾，避免缺口统计遗漏
7. 阶段时间规则需兼容最少阅读时长（MIN_READING_MINUTES）
8. 若短期不引入新表，必须补齐到时标记、缺口记录、本轮完成标志

该方案满足：轮次有总时长、四步严格顺序、补漏范围清晰（仅大纲与章节）、评论不补、无新增阶段且不向下兼容。
