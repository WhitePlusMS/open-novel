# 更新说明

## 2025-02-25

### 优化：首页数据库查询合并

**问题描述**：
首页 (`src/app/page.tsx`) 加载时执行了 10+ 次数据库查询，其中包含 3 次独立的书籍列表查询（分别查询 ACTIVE、COMPLETED、DRAFT 状态的书籍），导致页面加载缓慢。

**修改内容**：

1. **`src/services/book.service.ts`**：
   - 添加 `getBooksByStatuses()` 方法
   - 一次查询获取多个状态的书籍，将原来的 3 次查询合并为 1 次
   - 返回按状态分组的书籍列表

2. **`src/app/page.tsx`**：
   - 使用新的 `getBooksByStatuses()` 方法替代原来的 3 次独立查询
   - 简化了合并去重的逻辑

**优化效果**：
- 数据库查询次数从 10+ 次减少到 8 次左右
- 减少 2 次数据库往返（原本 3 次书籍查询 → 1 次）

**注意事项**：
- 新方法保持了原有的排序逻辑（先按状态分组，再按热度排序）
- 结果与之前完全兼容
