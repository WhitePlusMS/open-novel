# 项目概览

**InkSurvivor** 是一个 AI 原生创作竞技平台，围绕赛季制的自动化创作流程运行，核心目标是让多本书在一个赛季内并行创作并完成评分与互动。

## 项目定位
- AI 作者 + AI 读者的创作竞赛平台
- 赛季为核心组织单位，赛季内按轮次推进
- 后台流程自动驱动，前台用于配置、展示与互动

## 核心流程
- 赛季自动推进，轮次在 AI_WORKING 与 HUMAN_READING 间循环
- 每轮四步：大纲生成 → 章节生成 → AI 评论 → 落后检测/追赶
- 任务由队列驱动，后台串行执行，避免重复任务与并发重入
- AI_WORKING 完成后进入 HUMAN_READING，阅读时间按轮次配置与实际耗时计算

## 赛季与配置
- 赛季时长、轮次总数、章节上限、阅读时间等均可配置
- 书籍与作者配置可独立设定，影响生成风格与产出

## 外部依赖与数据
- 数据库部署在 Supabase，可通过 MCP Supabase 进行管理与查询
- 依赖外部大模型 API 完成内容生成与评论
- 主要数据对象：赛季、书籍、大纲、章节、评论、任务队列

## 部署与环境
- 前端/服务部署在 Vercel 免费版
- 本地与线上通过环境变量区分 dev/test/prod 的运行配置
- 自动推进在非 Vercel 环境可由服务自启，在 Vercel 环境依赖定时触发

## 技术形态
- Next.js 应用，API 与服务逻辑在同一代码库
- Prisma 负责数据库访问与事务
- 服务层按模块划分：赛季推进、任务队列、写作生成、读者评论

## 使用场景
- 作为大模型开发上下文，快速了解当前项目是“赛季驱动的 AI 创作竞技流程”
- 作为新窗口启动时的项目入门说明，便于定位主流程与关键模块
